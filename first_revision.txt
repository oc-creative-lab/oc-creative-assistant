## 二、目标产品形态
启动应用
↓
HomeLanding (首页,只有两个大按钮)
├─ [💬 聊天] → ChatEntry(选已有项目 / 新建项目) → ChatWorkspace
└─ [📚 项目库] → ProjectLibrary(项目卡片列表) → Workspace
ChatWorkspace 功能:

Chat Agent A 主动抛问题(由 Question Planner 反向指导)
用户自由说想法
Structured Agent B 后台异步抽实体/关系/字段
抽出的内容 → AgentStagingORM(暂存) → 工作台审阅落库
用户可随时退出 → 回首页

Workspace (工作台,/workspace/:projectId/...):
左栏 (WorkspaceSidebar):
- 项目名 / 简介
- 故事线        ──→ PlotCanvas (Vue Flow,只渲染 PlotNode)
- 角色卡        ──→ CharacterCardList (Notion 风格卡片网格)
└─ /:charId → CharacterCardDetail (自由字段编辑)
- 世界观        ──→ WorldCanvas (Vue Flow,只渲染 WorldNode)
三类内容之间自动建立关联(跨 sub-graph edge)
可召唤 agent 协助(FloatingChatDock 在工作台保留)

## 三、关键架构决策(必须严格遵守)

### 决策 1:一个 Project 下三个独立 sub-graph
ProjectORM:
id, name, description
plot_graph_id     (FK → GraphORM)
character_graph_id (FK → GraphORM)
world_graph_id    (FK → GraphORM)
GraphORM (新增):
id, project_id, section enum('plot'|'character'|'world')
NodeORM:
graph_id (FK,强化) — 节点归属到具体 sub-graph

### 决策 2:角色卡复用 CharacterNode 数据,新视图渲染

- CharacterGraph 在数据层仍是 graph(有 nodes + edges)
- 前端**不用 Vue Flow**,改用 CharacterCardList(Notion 风格网格)
- 角色卡"自由字段"放进 CharacterNode 的 metadata JSON
- 角色之间的关系边(师父/敌人/阵营)用卡片上的标签可视化,**不画线**

### 决策 3:独立 ProjectSeedORM 表
ProjectSeedORM (新增):
id, project_id (FK), version (int),
seed_json (text),
source enum('chat_end' | 'user_edit'),
created_at

### 决策 4:Chat Agent 启动注入 = Seed + RAG + 最近 N 轮

- 不喂完整历史对话(成本不可控)
- 启动时拼装:Project Seed (~500 tokens) + RAG top-3 (~1k tokens) + 最近 10 轮短期记忆 (~1k tokens)
- 总 context 控制在 2.5-3k tokens

### 决策 5:两个 LLM Agent 协作模式

- **Chat Agent A**(对用户):流式回复 + 自然抛问题
- **Structured Agent B**(后台):
  - `structured_extractor`:抽实体/关系/字段建议 → AgentStagingORM
  - `question_planner`:反向指导 A 下一步该问什么(注入 A 的 prompt)
- A 和 B 在 LangGraph 中**并行执行**,B 的结果不阻塞 A 的回复

### 决策 6:引入 Pinia(替换跨路由的 composables 共享)

- 现有 composables 不删,继续负责单画布内细粒度操作
- Pinia 新增:`useProjectStore` / `useGraphStore` / `useChatStore` / `useLibraryStore`
