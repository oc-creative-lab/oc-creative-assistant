## II. Goal of This Task

Rework the workbench's interaction experience from a "tool-style UI" into a "writing-friendly UI". Three specific changes:

### Change A: Canvas interactions better suited to writing habits
- **Right-click on empty canvas** → pop-up menu → choose a node type → the node is created at the mouse position
- **Right-click on a node** → pop-up menu (edit details / duplicate / delete / copy to chat box)
- A node's **title and summary** support **single-click inline edit**, with no need to switch to the right panel (the writing flow is not interrupted)
- **Double-click a node** → enter the "node detail page", which **fully takes over the center canvas area** (the canvas is hidden); the left and right panels stay unchanged, and there is a "← Back to Canvas" button in the top-right of the detail page

### Change B: AI assistant fixed at the bottom + the right panel becomes an AI output area
- Retire the FloatingChatDock floating mode; replace it with a **persistent single-line chat input box at the bottom** (similar to Cursor's bottom bar)
- Above the input box there is a **quoted-node card area** (select a node → copy to chat box → shown as a small dismissible card)
- **The right panel changes from a "node detail panel" to an "AI output display area"** (since details now live in the center)
- Right panel content: a **card stream (scrolling in reverse-chronological order)**, with card types including:
  - 🌐 web search result text
  - 📚 information blocks recalled by RAG
  - 💡 inspiration questions raised proactively by the AI
  - ✨ the AI's feedback / encouragement / praise
- Each card has a source-type tag + is collapsible
- The workbench's AI is an **independent lightweight "inspiration agent"**; it does not reuse ChatWorkspace's question_planner; it responds passively and does not force the creation forward

### Change C: Multi-select + copy to chat box
- Box-select / Shift+click to select multiple nodes (Vue Flow supports this natively; just enable it)
- After selecting, the right-click menu or the Ctrl+C shortcut → adds the nodes' `name + brief` as **quote cards** above the bottom input box
- The user can keep adding their own words in the input box → send to the AI (when the backend assembles the request, it includes the full data of the quoted nodes)

## V. Pinia Store Design

Add / extend the following stores:

```typescript
// stores/useCenterStageStore.ts
export const useCenterStageStore = defineStore('centerStage', {
  state: () => ({
    mode: 'canvas' as 'canvas' | 'detail',
    detailNodeId: null as string | null,
  }),
  actions: {
    openDetail(nodeId: string) {
      this.mode = 'detail'
      this.detailNodeId = nodeId
    },
    returnToCanvas() {
      this.mode = 'canvas'
      this.detailNodeId = null
    },
  },
})

// stores/useComposerStore.ts
interface QuotedNodeRef {
  id: string
  type: string  // 'character' | 'plot' | 'world'
  title: string
}

export const useComposerStore = defineStore('composer', {
  state: () => ({
    references: [] as QuotedNodeRef[],
    input: '',
  }),
  actions: {
    addReferences(refs: QuotedNodeRef[]) {
      // dedupe
      const existing = new Set(this.references.map(r => r.id))
      refs.forEach(r => { if (!existing.has(r.id)) this.references.push(r) })
    },
    removeReference(id: string) {
      this.references = this.references.filter(r => r.id !== id)
    },
    clear() {
      this.references = []
      this.input = ''
    },
  },
})

// stores/useAiOutputStore.ts
interface AiOutput {
  id: string
  type: 'search' | 'rag' | 'question' | 'feedback'
  content: string
  timestamp: number
  collapsed: boolean
  metadata?: Record<string, any>  // e.g. the url for search, the source_node_id for rag
}

export const useAiOutputStore = defineStore('aiOutput', {
  state: () => ({
    outputs: [] as AiOutput[],
  }),
  actions: {
    push(output: Omit<AiOutput, 'id' | 'timestamp' | 'collapsed'>) {
      this.outputs.unshift({
        ...output,
        id: crypto.randomUUID(),
        timestamp: Date.now(),
        collapsed: false,
      })
    },
    toggleCollapse(id: string) {
      const o = this.outputs.find(x => x.id === id)
      if (o) o.collapsed = !o.collapsed
    },
    clear() { this.outputs = [] },
  },
})
```

## VI. Implementation Notes for the Four Key Interactions

### 1. Inline Edit (contenteditable within a node)

`InlineEditableText.vue`:

```vue
<template>
  <span
    v-if="!editing"
    class="inline-text"
    @click="enterEdit"
  >{{ modelValue || placeholder }}</span>
  <input
    v-else
    ref="inputEl"
    v-model="draft"
    class="inline-input nodrag nopan"
    @keydown.enter.prevent="save"
    @keydown.esc.prevent="cancel"
    @blur="save"
  />
</template>

<script setup lang="ts">
import { ref, nextTick, watch } from 'vue'
const props = defineProps<{ modelValue: string; placeholder?: string }>()
const emit = defineEmits<{ 'update:modelValue': [string]; save: [string] }>()

const editing = ref(false)
const draft = ref('')
const inputEl = ref<HTMLInputElement>()

const enterEdit = async () => {
  draft.value = props.modelValue
  editing.value = true
  await nextTick()
  inputEl.value?.focus()
  inputEl.value?.select()
}

const save = () => {
  if (draft.value !== props.modelValue) {
    emit('update:modelValue', draft.value)
    emit('save', draft.value)
  }
  editing.value = false
}

const cancel = () => {
  draft.value = props.modelValue
  editing.value = false
}
</script>
```

**Key constraints**:
- The edit-mode `<input>` must have `class="nodrag nopan"`, otherwise Vue Flow will intercept mouse events
- The node component's root div must also keep its original class names such as `class="vue-flow__node-default"`

### 2. Right-Click Menu

In `CanvasWorkspace.vue`, listen to the hooks provided by Vue Flow:

```typescript
import { useVueFlow } from '@vue-flow/core'

const { onPaneContextMenu, onNodeContextMenu, addNodes, project } = useVueFlow()
const contextMenu = ref<{ show: boolean; x: number; y: number; type: 'blank' | 'node'; node?: any }>({
  show: false, x: 0, y: 0, type: 'blank'
})

onPaneContextMenu(({ event }) => {
  event.preventDefault()
  contextMenu.value = { show: true, x: event.clientX, y: event.clientY, type: 'blank' }
})

onNodeContextMenu(({ event, node }) => {
  event.preventDefault()
  contextMenu.value = { show: true, x: event.clientX, y: event.clientY, type: 'node', node }
})

const handleCreateNode = (nodeType: string) => {
  // Infer the default type from the current sub-graph's section, but also allow choosing from the menu
  const flowPos = project({ x: contextMenu.value.x, y: contextMenu.value.y })
  addNodes([nodeFactory.create(nodeType, flowPos)])
  contextMenu.value.show = false
}
```

`CanvasContextMenu.vue` renders different menu items based on `menuType`:
- `blank`: "Create a [current section] node" (which type to show is decided by useProjectStore().currentSection)
- `node`: edit details (open NodeDetailView) / duplicate / delete / copy to chat box

### 3. Multi-Select + Copy to Chat Box

Vue Flow supports multi-select natively; just enable it:

```typescript
// CanvasWorkspace.vue
import { useVueFlow } from '@vue-flow/core'
const { getSelectedNodes } = useVueFlow()

const copySelectedToComposer = () => {
  const composer = useComposerStore()
  const refs = getSelectedNodes.value.map(n => ({
    id: n.id,
    type: n.type,
    title: n.data.title || n.data.name || 'Untitled',
  }))
  composer.addReferences(refs)
}

// Listen for keyboard events (when the canvas is focused)
const onKeyDown = (e: KeyboardEvent) => {
  if ((e.ctrlKey || e.metaKey) && e.key === 'c' && getSelectedNodes.value.length > 0) {
    e.preventDefault()
    copySelectedToComposer()
  }
}
```

Vue Flow's selectionKeyCode defaults to `Shift`, and box-select defaults to `selectionMode="partial"`. Check and enable them:

```vue
<VueFlow

  selection-mode="partial"
  multi-selection-key-code="['Shift', 'Meta', 'Control']"
  ...
/>
```

## VII. Backend Changes (Lightweight, Mainly the Source of the Right Panel's AI Output)

### New LangGraph node: `workspace_inspiration_agent`

**Different from ChatWorkspace's question_planner**:
- It does not proactively push the creation forward; it only responds when the user actively invokes it (the user sends a message with quoted nodes in the bottom chat box)
- It outputs multiple types: it could be a RAG recall, a search result, an inspiration question, or feedback
- It pushes to the frontend via SSE, and the frontend dispatches it by type to the RightStageOutput cards

Files:
- `app/agents/nodes/workspace_inspiration.py`
- `app/agents/prompts/workspace_inspiration.v1.md`

Simplified logic:
```python
def workspace_inspiration_node(state):
    user_msg = state.user_message
    quoted_nodes = state.quoted_nodes  # the quoted-node data passed from the frontend

    # Decide the output type
    intent = classify_intent(user_msg)
    # 'need_info' → RAG / search
    # 'stuck' → question
    # 'showing_off' → feedback

    if intent == 'need_info':
        results = rag_service.hybrid_retrieve(user_msg, ...)
        yield {'type': 'rag', 'content': format_results(results)}
        # Trigger web_search_client when needed (reuse the existing one)
    elif intent == 'stuck':
        question = generate_inspiration_question(state)
        yield {'type': 'question', 'content': question}
    elif intent == 'showing_off':
        feedback = generate_feedback(state)
        yield {'type': 'feedback', 'content': feedback}
```

### New API: `/api/projects/{id}/workspace_chat`

Similar to the existing `/api/chat`, but dedicated to the workbench:
- Receives `{ message, quoted_node_ids[] }`
- Pushes AI output via SSE streaming
- Each output carries a `type` field: `search` / `rag` / `question` / `feedback`
- The frontend `RightStageOutput.vue` listens to the SSE, and pushes each received item into `useAiOutputStore`

## IX. Strict Execution Constraints

1. **Reuse first**: node components, Vue Flow, the SSE stream, and the RAG service are all reused as-is; don't reinvent the wheel
2. **Don't break ChatWorkspace**: this task only changes the workbench; ChatWorkspace is untouched
3. **Inline edit must resolve the Vue Flow drag conflict**: add the `nodrag nopan` class to all contenteditable elements
4. **The right-click menu's z-index must be above nodes and the canvas**: use `position: fixed` + `z-index: 9999`
5. **When double-clicking a node to enter the detail page**: don't let a single-click inline edit accidentally trigger the detail page (single click = inline edit, double click = enter detail; Vue Flow's `onNodeClick` and `onNodeDoubleClick` handle them separately)
6. **The bottom chat box is single-line**: Enter sends, no Shift+Enter newline (keep it simple); use quoted nodes instead of pasting very long text
7. **AI output cards are "in reverse-chronological order"**: the newest at the top, older ones sinking down
8. **The workbench agent is a passive agent**: it doesn't speak proactively unless the user sends a message (distinguishing it from ChatWorkspace's proactive-questioning agent)
