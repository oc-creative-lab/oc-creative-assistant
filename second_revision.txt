## 二、本任务的目标

把工作台的交互体验从"工具型 UI"重做成"写作友好型 UI"。具体三大改点:

### 改点 A:画布交互更符合写作习惯
- 画布**空白处右键** → 弹出菜单 → 选节点类型 → 节点在鼠标位置生成
- **节点上右键** → 弹出菜单(编辑详情 / 复制 / 删除 / 复制到对话框)
- 节点上的**标题和简介**支持**单击 inline edit**,无需切右栏(写作流程不打断)
- **双击节点** → 进入"节点详情页",**完全占据中栏画布位置**(画布隐藏),左右栏保持不变,详情页右上角有"← 返回画布"按钮

### 改点 B:AI 助手底部固定 + 右栏变 AI 输出区
- 废弃 FloatingChatDock 浮动模式,改为**底部常驻的单行对话输入框**(类似 Cursor 底部)
- 输入框上方有**引用节点卡片区**(选中节点 → 复制到对话框 → 显示为可叉掉的小卡片)
- **右栏从"节点详情面板"改为"AI 输出展示区"**(因为详情现在到中栏了)
- 右栏内容:**卡片流(按时间线倒序滚动)**,卡片类型包括:
  - 🌐 联网搜索结果文本
  - 📚 RAG 召回的信息块
  - 💡 AI 主动提出的灵感问题
  - ✨ AI 的反馈 / 鼓励 / 夸夸
- 每张卡片有来源类型标签 + 可折叠
- 工作台的 AI 是**独立的轻量"灵感 agent"**,不复用 ChatWorkspace 的 question_planner,被动响应、不强制推进创作

### 改点 C:多选 + 复制到对话框
- 框选 / Shift+点击 选中多个节点(Vue Flow 原生支持,启用即可)
- 选中后,右键菜单或快捷键 Ctrl+C → 把节点的 `name + brief` 以**引用卡片**形式加到底部输入框上方
- 用户可继续在输入框内加自己的话 → 发送给 AI(后端拼装时把引用节点的完整数据带上)

## 五、Pinia store 设计

新增 / 扩展以下 store:

```typescript
// stores/useCenterStageStore.ts
export const useCenterStageStore = defineStore('centerStage', {
  state: () => ({
    mode: 'canvas' as 'canvas' | 'detail',
    detailNodeId: null as string | null,
  }),
  actions: {
    openDetail(nodeId: string) {
      this.mode = 'detail'
      this.detailNodeId = nodeId
    },
    returnToCanvas() {
      this.mode = 'canvas'
      this.detailNodeId = null
    },
  },
})

// stores/useComposerStore.ts
interface QuotedNodeRef {
  id: string
  type: string  // 'character' | 'plot' | 'world'
  title: string
}

export const useComposerStore = defineStore('composer', {
  state: () => ({
    references: [] as QuotedNodeRef[],
    input: '',
  }),
  actions: {
    addReferences(refs: QuotedNodeRef[]) {
      // 去重
      const existing = new Set(this.references.map(r => r.id))
      refs.forEach(r => { if (!existing.has(r.id)) this.references.push(r) })
    },
    removeReference(id: string) {
      this.references = this.references.filter(r => r.id !== id)
    },
    clear() {
      this.references = []
      this.input = ''
    },
  },
})

// stores/useAiOutputStore.ts
interface AiOutput {
  id: string
  type: 'search' | 'rag' | 'question' | 'feedback'
  content: string
  timestamp: number
  collapsed: boolean
  metadata?: Record<string, any>  // 比如 search 的 url、rag 的 source_node_id
}

export const useAiOutputStore = defineStore('aiOutput', {
  state: () => ({
    outputs: [] as AiOutput[],
  }),
  actions: {
    push(output: Omit<AiOutput, 'id' | 'timestamp' | 'collapsed'>) {
      this.outputs.unshift({
        ...output,
        id: crypto.randomUUID(),
        timestamp: Date.now(),
        collapsed: false,
      })
    },
    toggleCollapse(id: string) {
      const o = this.outputs.find(x => x.id === id)
      if (o) o.collapsed = !o.collapsed
    },
    clear() { this.outputs = [] },
  },
})
```

## 六、四个关键交互的实现要点

### 1. Inline Edit(节点内 contenteditable)

`InlineEditableText.vue`:

```vue
<template>
  <span
    v-if="!editing"
    class="inline-text"
    @click="enterEdit"
  >{{ modelValue || placeholder }}</span>
  <input
    v-else
    ref="inputEl"
    v-model="draft"
    class="inline-input nodrag nopan"
    @keydown.enter.prevent="save"
    @keydown.esc.prevent="cancel"
    @blur="save"
  />
</template>

<script setup lang="ts">
import { ref, nextTick, watch } from 'vue'
const props = defineProps<{ modelValue: string; placeholder?: string }>()
const emit = defineEmits<{ 'update:modelValue': [string]; save: [string] }>()

const editing = ref(false)
const draft = ref('')
const inputEl = ref<HTMLInputElement>()

const enterEdit = async () => {
  draft.value = props.modelValue
  editing.value = true
  await nextTick()
  inputEl.value?.focus()
  inputEl.value?.select()
}

const save = () => {
  if (draft.value !== props.modelValue) {
    emit('update:modelValue', draft.value)
    emit('save', draft.value)
  }
  editing.value = false
}

const cancel = () => {
  draft.value = props.modelValue
  editing.value = false
}
</script>
```

**关键约束**:
- 编辑态的 `<input>` 必须加 `class="nodrag nopan"`,否则 Vue Flow 会拦截鼠标事件
- 节点组件的根 div 也要保留 `class="vue-flow__node-default"` 等原有类名

### 2. 右键菜单

在 `CanvasWorkspace.vue` 内监听 Vue Flow 提供的 hook:

```typescript
import { useVueFlow } from '@vue-flow/core'

const { onPaneContextMenu, onNodeContextMenu, addNodes, project } = useVueFlow()
const contextMenu = ref<{ show: boolean; x: number; y: number; type: 'blank' | 'node'; node?: any }>({
  show: false, x: 0, y: 0, type: 'blank'
})

onPaneContextMenu(({ event }) => {
  event.preventDefault()
  contextMenu.value = { show: true, x: event.clientX, y: event.clientY, type: 'blank' }
})

onNodeContextMenu(({ event, node }) => {
  event.preventDefault()
  contextMenu.value = { show: true, x: event.clientX, y: event.clientY, type: 'node', node }
})

const handleCreateNode = (nodeType: string) => {
  // 根据当前 sub-graph 的 section 推断默认类型,但也支持从菜单选
  const flowPos = project({ x: contextMenu.value.x, y: contextMenu.value.y })
  addNodes([nodeFactory.create(nodeType, flowPos)])
  contextMenu.value.show = false
}
```

`CanvasContextMenu.vue` 根据 `menuType` 渲染不同菜单项:
- `blank`: "新建 [当前 section] 节点"(根据 useProjectStore().currentSection 决定显示哪种)
- `node`: 编辑详情(打开 NodeDetailView)/ 复制 / 删除 / 复制到对话框

### 3. 多选 + 复制到对话框

Vue Flow 原生支持多选,启用即可:

```typescript
// CanvasWorkspace.vue
import { useVueFlow } from '@vue-flow/core'
const { getSelectedNodes } = useVueFlow()

const copySelectedToComposer = () => {
  const composer = useComposerStore()
  const refs = getSelectedNodes.value.map(n => ({
    id: n.id,
    type: n.type,
    title: n.data.title || n.data.name || '未命名',
  }))
  composer.addReferences(refs)
}

// 监听键盘事件(在画布 focus 时)
const onKeyDown = (e: KeyboardEvent) => {
  if ((e.ctrlKey || e.metaKey) && e.key === 'c' && getSelectedNodes.value.length > 0) {
    e.preventDefault()
    copySelectedToComposer()
  }
}
```

Vue Flow 的 selectionKeyCode 默认是 `Shift`,框选默认开启 `selectionMode="partial"`。检查并启用:

```vue
<VueFlow
  :selection-key-code="['Shift']"
  selection-mode="partial"
  multi-selection-key-code="['Shift', 'Meta', 'Control']"
  ...
/>
```

## 七、后端改动(轻量,主要是右栏 AI 输出的来源)

### 新增 LangGraph 节点:`workspace_inspiration_agent`

**与 ChatWorkspace 的 question_planner 不同**:
- 不主动推进创作,只在用户主动调用时响应(用户在底部对话框发了带引用节点的消息)
- 输出多种类型:可能是 RAG 召回、可能是搜索结果、可能是灵感问题、可能是反馈
- 通过 SSE 推送给前端,前端按类型分发到 RightStageOutput 卡片

文件:
- `app/agents/nodes/workspace_inspiration.py`
- `app/agents/prompts/workspace_inspiration.v1.md`

简化逻辑:
```python
def workspace_inspiration_node(state):
    user_msg = state.user_message
    quoted_nodes = state.quoted_nodes  # 从前端传来的引用节点数据
    
    # 决定输出类型
    intent = classify_intent(user_msg)  
    # 'need_info' → RAG / search
    # 'stuck' → question
    # 'showing_off' → feedback
    
    if intent == 'need_info':
        results = rag_service.hybrid_retrieve(user_msg, ...)
        yield {'type': 'rag', 'content': format_results(results)}
        # 必要时触发 web_search_client(沿用现有)
    elif intent == 'stuck':
        question = generate_inspiration_question(state)
        yield {'type': 'question', 'content': question}
    elif intent == 'showing_off':
        feedback = generate_feedback(state)
        yield {'type': 'feedback', 'content': feedback}
```

### 新增 API:`/api/projects/{id}/workspace_chat`

类似现有 `/api/chat`,但是为工作台专用:
- 接收 `{ message, quoted_node_ids[] }`
- 通过 SSE 流式推送 AI 输出
- 每条输出带 `type` 字段:`search` / `rag` / `question` / `feedback`
- 前端 `RightStageOutput.vue` 监听 SSE,收到一条就 push 进 `useAiOutputStore`

## 九、严格执行约束

1. **复用优先**:节点组件、Vue Flow、SSE 流、RAG 服务全部沿用现有,不另起炉灶
2. **不破坏 ChatWorkspace**:本任务只改工作台,ChatWorkspace 不动
3. **inline edit 必须解决 Vue Flow 拖拽冲突**:所有 contenteditable 元素加 `nodrag nopan` 类
4. **右键菜单要 z-index 高于节点和画布**:用 `position: fixed` + `z-index: 9999`
5. **节点双击进详情页时**:不要让单击 inline edit 误触发详情页(单击 = inline edit,双击 = 进详情;Vue Flow 的 `onNodeClick` 和 `onNodeDoubleClick` 分别处理)
6. **底部对话框是单行**:Enter 发送,无 Shift+Enter 换行(保持简单);超长文本用引用节点而非粘贴
7. **AI 输出卡片"按时间线倒序"**:最新的在最上面,旧的往下沉
8. **工作台 agent 是被动 agent**:用户不发消息它不主动说话(与 ChatWorkspace 的主动提问 agent 区分)
